Vura Mineral Skincare · Design System

VURA

A herbarium-green, paper-and-mineral design system for a skincare brand rooted in Indian botany. v0.2.0 — foundations pass.

What's new in 0.2.0 — Single self-contained portal (no CORS), inline-SVG wordmark + seal, W3C tokens.json, focus-ring + contrast tokens, motion sandbox, paper-first surface mode, versioning.

Foundations

Brand & Voice

Vura is the Hindi root meaning earth that heals. The brand sits in one dark, herbarium-like green; its accents come from saffron mustard, terracotta clay, and burgundy petal. The system is paper-first — like the printed label on the bottle — with green as a brand block, not a canvas.

Voice

Quiet, grounded, generous. Like a senior pharmacist who has been formulating for thirty years and has nothing to prove.

We say
  • "Mineral-rich phitkari for sensitive skin."
  • "Made slowly. In Hyderabad."
  • "Aloe and rose, nothing else."
We do not say
  • "Revolutionary." / "Game-changer." / "Disrupt."
  • "Glow up." / "Clean girl." / Internet trends.
  • Any superlative without a number to back it.

Surface principles

Vura is paper-first. The dark green is a brand block — reserved for heroes, product cards, callouts, the wordmark plate. The bulk of the experience reads as label paper.

Accessibility

Vura targets WCAG 2.1 AA for all consumer surfaces. Body text ≥ 4.5:1 contrast; large text (18px bold, 24px regular) ≥ 3:1; focus visible on every interactive element.

Focus ring · tokens

Two rings — one for paper, one for brand block. Both are 2px solid with a 3px offset; they pick up the accent that contrasts with the surface they're on.

--focus-ring-on-paper · #8D3B2D (terracotta)
--focus-ring-on-block · #FFC03F (saffron)
:focus-visible {
  outline: var(--focus-ring-width) solid var(--focus-ring-on-paper);
  outline-offset: var(--focus-ring-offset);
}
.on-block :focus-visible { outline-color: var(--focus-ring-on-block); }

UI Icons

A locked subset of Lucide. These are the only utility icons permitted in Vura UI. Adding a new one requires a minor version bump and a CHANGELOG entry. All inline SVG, stroke-width 1.5, 24 × 24 viewBox — visually consistent with the bespoke pillar set.

Audit response. Pillar illustrations and UI icons used to live on different stroke systems (bespoke thin-line vs Lucide). They're now visually paired: same 1.5px stroke, same line caps, same optical sizing.

Adding a new icon

  1. Open a PR with the SVG inline (24 × 24, stroke 1.5, currentColor).
  2. Visually pair with the closest existing icon — same line caps, same optical weight.
  3. Add a row to this page (static HTML — no array, no loop).
  4. Bump minor version in tokens.json & add CHANGELOG entry.

Buttons

Five buttons. Press is the new primary commerce CTA — saffron with a flat 2px terracotta relief shadow, reserved for Add to cart, Buy now, Checkout and other commit actions. btn-primary remains for in-flow secondary commits; block and ghost variants are unchanged.

Press · primary commerce CTA

Solid --vura-saffron fill, --vura-forest-deep ink, sharp 2px --vura-terracotta drop-shadow offset bottom-right with zero blur. On :active the button translates 2px down-right and the shadow collapses — the surface visibly presses into the page.

Generating CSS / iOS / Android

This file conforms to the W3C DTCG specification. Feed it to Style Dictionary to emit:

  • tokens.css — ready-to-import CSS custom properties (already shipped inline in this portal as :root { … }).
  • tokens.scss — for Sass-based codebases.
  • VuraTokens.swift — UIColor + CGFloat constants for iOS.
  • colors.xml + dimens.xml — for Android.

The Style Dictionary build pipeline itself is deferred to 0.3.0 — the hand-written JSON is the single source of truth for now.
